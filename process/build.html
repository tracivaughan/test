Building is centered on implementing the designs developed in Design and Ideation. Depending on the teamâ€™s selected methodology, this stage can include multiple time-boxed sprints or phases, with individual sections or features being designed, built, and tested. Continual deployment to test environments allows features to be reviewed, and progress to be marked. The most essential features (as defined in Design and Ideation) are created and evaluated first.

A number of roles can be involved in the Build phase, depending on the project type (mobile or web application, or batch? ), scale (major overhaul, or small update/work request?), author of the project (in-house solution, or vended product implementation?), and context (new application, or replacement application where data may need to be converted?). Whatever the product type, this phase is where the product vision is turned into executable software.

Project activities

  • Programming
  • Deployment to test environments
  • Database design & modeling, table and other db object creation
  • Data conversion/migration
  • Database query implementation/optimization
  • Testing-as-you-go (stakeholder reviews, usability tests, accessibility checks)
  • Create communication plans and training materials
  • Build AppWorx process flows and their components
  • Applications directory creation/updates

Deliverables and outcomes

  • Builds ready for testing and deployment
  • Updated product specifications
  • Updated testing plan, reflecting building-stage adjustments and changes
  • Database object setup and data migration scripts
  • AppWorx process flows and their components ready for testing and deployment
